# 基于OpenHarmony的智慧农业控制系统

# 软件系统设计文档

---

## 1 引言

### 1.1 编写目的

本文档是"基于OpenHarmony的智慧农业控制系统"的软件系统设计说明书，旨在对系统的总体架构、模块划分、接口定义、数据库设计等进行详细描述，为开发人员提供编码实现的技术依据，为测试人员提供测试设计的参考。

### 1.2 项目背景

本项目为第十七届中国大学生服务外包创新创业大赛A类赛题（A03），由诚迈科技股份有限公司命题。系统基于OpenHarmony操作系统开发智慧农业控制APP，配合Python后端服务，实现农业环境远程监测与智能控制。

## 2 程序系统设计概述

### 2.1 业务需求

系统面向农业种植场景，核心业务需求如下：

（1）农户需要远程实时查看大棚/田地的环境参数（温度、湿度、光照、CO₂、土壤湿度），以便及时了解农作物生长环境状况。

（2）农户需要远程控制灌溉设备（水泵、水阀）和补光设备（LED灯），减少人工到场操作的频率。

（3）系统需要根据环境参数自动执行设备联动控制（如土壤干燥自动灌溉、光照不足自动补光），降低人工干预成本。

（4）系统需要在环境参数异常时自动触发预警，通知农户及时处理。

（5）管理员需要查看历史数据报告，分析环境趋势，优化种植策略。

### 2.2 功能需求

根据业务需求分析，系统功能需求归纳为以下七大模块：

| 模块编号 | 模块名称 | 功能概述 |
|---------|---------|---------|
| M01 | 用户认证 | 用户注册、登录、Token认证 |
| M02 | 智能控制中心 | 设备管理、指令转发、联动配置、系统概览 |
| M03 | 智能灌溉 | 土壤湿度监测、灌溉量计算、水泵水阀控制、用水统计 |
| M04 | 智能光照 | 光照监测、LED亮度调节、补光建议 |
| M05 | 环境监测 | 五参数实时采集、可视化展示、历史数据查询 |
| M06 | 预警管理 | 阈值规则配置、预警触发、预警日志 |
| M07 | 扩展功能 | 通风风扇接入、转速控制、CO₂联动 |

### 2.3 非功能需求

（1）**稳定性**：系统7×24小时运行，数据采集间隔≤5秒，WebSocket断线自动重连。

（2）**响应速度**：页面加载≤2秒，设备控制响应≤1秒，API平均响应≤500ms。

（3）**安全性**：密码哈希存储，Token认证，CORS安全策略。

（4）**可扩展性**：设备类型动态扩展，规则动态配置，模块化低耦合设计。

### 5.2 服务端系统设计

服务端采用FastAPI框架，按照分层架构组织代码：

```
backend/
├── app/
│   ├── main.py              # 应用入口、生命周期管理、模拟循环

│   ├── database.py          # 数据库引擎与会话管理
│   ├── models.py            # SQLAlchemy ORM模型（12个实体类）
│   ├── schemas.py           # Pydantic请求/响应数据校验模型
│   ├── websocket.py         # WebSocket连接管理器（广播机制）
│   ├── routers/             # API路由层（13个路由模块）
│   │   ├── auth.py          # POST /api/auth/login, /register
│   │   ├── sensor.py        # GET  /api/sensor/latest, /history, /stats
│   │   ├── device.py        # GET  /api/device/list, POST /control, PUT /params
│   │   ├── alert.py         # GET  /api/alert/rules, /logs, PUT /rules/{id}
│   │   ├── automation.py    # CRUD /api/automation
│   │   ├── schedule.py      # CRUD /api/schedule
│   │   ├── zone.py          # CRUD /api/zone
│   │   ├── crop.py          # CRUD /api/crop
│   │   ├── report.py        # GET  /api/report/list, /daily
│   │   ├── water.py         # GET  /api/water/today, /stats
│   │   ├── weather.py       # GET  /api/weather/current
│   │   ├── camera.py        # GET  /api/camera/snapshot
│   │   └── system.py        # GET  /api/system/overview
```

## 8 各模块详细设计

### 8.1 用户认证模块（M01）

**功能描述**：提供用户注册和登录功能，基于Token的身份认证机制。

**后端设计**：
- 路由文件：`routers/auth.py`
- 注册接口：接收用户名和密码，密码经哈希处理后存入user表，返回Token
- 登录接口：验证用户名和密码哈希，验证通过返回Token、用户名、角色

**前端设计**：
- 页面文件：`pages/Login.ets`
- 登录/注册Tab切换，TextInput输入框，Button提交
- 登录成功后通过AppStorage保存Token，router.replaceUrl跳转首页

**接口定义**：

| 接口 | 方法 | 路径 | 请求体 | 响应体 |
|------|------|------|--------|--------|
| 登录 | POST | /api/auth/login | {username, password} | {access_token, username, role} |
| 注册 | POST | /api/auth/register | {username, password} | {access_token, username, role} |

### 8.2 智能环境监测模块（M05）

**功能描述**：每5秒采集温度、湿度、光照、CO₂、土壤湿度五项参数，APP实时展示并支持历史查询。

**后端设计**：
- 路由文件：`routers/sensor.py`
- 服务文件：`services/simulator.py`（数据模拟生成器）
- 模拟器基于上一条数据进行随机波动，模拟真实传感器行为
- 新数据生成后通过WebSocket广播至所有连接的客户端

**前端设计**：
- 视图文件：`views/DashboardView.ets`（首页卡片展示）、`views/MonitorView.ets`（详细监测）
- DashboardView：概览卡片（设备总数/在线/预警）+ 五项传感器数据卡片 + 快捷操作入口
- MonitorView：实时数据面板（带颜色标识）+ 历史数据列表（分页加载）

### 8.3 设备控制模块（M02）

**功能描述**：管理5类设备（水阀、水泵、LED、摄像头、风扇），支持远程开关和参数调节。

**后端设计**：
- 路由文件：`routers/device.py`
- 设备控制：更新device表status字段，记录device_log
- 参数调节：更新device表params JSON字段（brightness/speed/flow_rate）

**前端设计**：
- 视图文件：`views/DeviceView.ets`（设备列表）
- 页面文件：`pages/DeviceDetail.ets`（设备详情）
- DeviceView：List + ForEach渲染设备卡片，Toggle开关控制，点击进入详情
- DeviceDetail：设备状态展示、Slider参数调节、操作日志列表

**接口定义**：

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 设备列表 | GET | /api/device/list | 返回所有设备及状态参数 |
| 设备开关 | POST | /api/device/{id}/control | body: {status: 0或1} |
| 参数调节 | PUT | /api/device/{id}/params | body: {params: {key: value}} |
| 操作日志 | GET | /api/device/log | 支持device_id筛选和分页 |

### 8.4 智能灌溉模块（M03）

**功能描述**：监测土壤湿度，自动计算灌溉量，远程控制水泵水阀，统计用水量。

**后端设计**：
- 灌溉量计算算法：`建议灌溉量 = (目标湿度 - 当前湿度) × 0.5`（单位：升）
- 用水量记录：水泵/水阀运行期间，每个采集周期按流量自动记录用水量
- 路由文件：`routers/water.py`（用水统计接口）

**前端设计**：
- 页面文件：`pages/IrrigationPage.ets`
- 土壤湿度大卡片（数值 + 状态评估 + 进度条）
- 目标湿度Slider + 灌溉量自动计算显示
- 水泵/水阀Toggle控制
- 今日用水量和流量统计

### 8.5 智能光照模块（M04）

**功能描述**：监测光照强度，精准调节LED补光灯亮度，提供补光建议。

**前端设计**：
- 页面文件：`pages/LightingPage.ets`
- 光照强度大卡片（数值 + 状态评估 + 进度条）
- LED开关Toggle + 亮度Slider（0-100%）+ 快捷按钮（25%/50%/75%/100%）
- 目标光照Slider + 补光建议文字（建议亮度百分比 = (目标光照-当前光照)/800）

### 8.6 预警管理模块（M06）

**功能描述**：为五项环境参数配置安全阈值范围，超出范围自动触发预警并记录。

**后端设计**：
- 服务文件：`services/alert_checker.py`
- 检测逻辑：遍历已启用的alert_rule，判断传感器数据是否超出[min_value, max_value]范围
- 触发时写入alert_log表，通过WebSocket推送预警消息

**前端设计**：
- 视图文件：`views/AlertView.ets`
- Tab切换：预警规则 / 预警记录
- 规则列表：显示指标名、阈值范围、Toggle启用开关
- 记录列表：显示预警类型（过高/过低）、指标值、时间、未读红点

### 8.7 联动规则模块

**功能描述**：配置传感器数据与设备动作的自动联动规则。

**后端设计**：
- 服务文件：`services/automation_engine.py`
- 执行逻辑：每次新数据生成后，遍历已启用的automation_rule，按条件（gt/lt/eq）判断是否触发
- 触发时自动执行设备动作（开启/关闭/设置参数），记录操作日志（source='auto'）

**默认联动规则**：

| 触发条件 | 目标设备 | 动作 |
|---------|---------|------|
| 土壤湿度 < 30% | 水泵 | 开启 |
| 土壤湿度 > 70% | 水泵 | 关闭 |
| 光照强度 < 5000 Lux | LED补光灯 | 开启（亮度80%） |
| CO₂浓度 > 1000 ppm | 通风风扇 | 开启（转速100%） |
| CO₂浓度 < 600 ppm | 通风风扇 | 关闭 |

### 8.8 扩展功能模块 - 通风风扇（M07）

**功能描述**：验证系统设备扩展能力，通风风扇通过标准设备接口接入系统。

**设计要点**：
- 设备类型标识：`fan`，可控参数：`speed`（0-100%）
- 复用现有device表结构和设备控制接口，无需修改核心架构
- 通过联动规则实现CO₂浓度与风扇的自动联动
- 验证结论：新设备仅需在init.sql中插入一条设备记录即可接入，证明系统扩展性良好

### 9.2 数据库物理结构

所有数据表使用InnoDB存储引擎，字符集utf8mb4，排序规则utf8mb4_unicode_ci。

**主要表的索引设计**：

| 表名 | 索引 | 类型 | 说明 |
|------|------|------|------|
| user | username | UNIQUE | 用户名唯一约束 |
| sensor_data | id | PRIMARY | 自增主键，按时间顺序 |
| device | id | PRIMARY | 设备主键 |
| alert_log | rule_id | FOREIGN KEY | 关联预警规则 |
| automation_rule | action_device_id | FOREIGN KEY | 关联目标设备 |
| zone_device | (zone_id, device_id) | UNIQUE | 防止重复关联 |
| daily_report | report_date | UNIQUE | 每日唯一报告 |
| water_usage | device_id | FOREIGN KEY | 关联灌溉设备 |

**初始数据**：
- 5条设备记录（水阀、水泵、LED补光灯、监控摄像头、通风风扇）
- 5条预警规则（温度、湿度、光照、CO₂、土壤湿度的安全范围）
- 5条联动规则（土壤湿度→水泵、光照→LED、CO₂→风扇）
- 2条区域记录（1号温室大棚、露天试验田）

### 9.3 WebSocket通信协议设计

**连接地址**：`ws://{host}:8000/ws/realtime`

**消息格式**（JSON）：

```json
{
  "type": "sensor_data | alert | device_status",
  "data": { ... }
}
```

**消息类型说明**：

| 类型 | 触发时机 | 数据内容 | 推送频率 |
|------|---------|---------|---------|
| sensor_data | 每个采集周期 | 最新传感器五项参数 | 每5秒 |
| alert | 预警触发时 | 预警详情列表 | 事件驱动 |
| device_status | 设备状态变化时 | 变化的设备信息 | 事件驱动 |
