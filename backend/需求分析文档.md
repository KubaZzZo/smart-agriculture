# 基于OpenHarmony的智慧农业控制系统

# 需求分析文档

---

## 1 引言

### 1.1 编写目的

本文档是"基于OpenHarmony的智慧农业控制系统"项目的需求分析说明书，旨在明确系统的功能需求、非功能需求及约束条件，为后续的系统设计、编码实现和测试验证提供依据。本文档的预期读者包括项目开发人员、测试人员、项目管理人员及评审专家。

### 1.2 项目背景

本项目来源于第十七届中国大学生服务外包创新创业大赛A类赛题（A03），由诚迈科技股份有限公司命题。

我国农业长期呈现"大国小农"的基本格局，产业发展受多重因素制约：农业从业人员数量不足、年龄结构偏老化，同时农业用地规模逐步缩减。智慧农业以信息和知识为核心驱动要素，通过互联网、物联网、大数据、人工智能及智能装备等现代信息技术与农业深度跨界融合，实现农业生产全流程的信息实时感知、定量科学决策、设备智能控制、资源精准投入及个性化服务。

本项目基于OpenHarmony操作系统，设计开发一款智慧农业控制APP，实现远程监测农作物环境的温度、湿度、光照、二氧化碳浓度及土壤湿度，并控制水阀、水泵、LED补光灯、通风风扇等设备，为农户带来更智能、更高效的农作物种植体验。

## 2 任务概述

### 2.1 项目目标

（1）开发一款基于OpenHarmony的智慧农业控制APP，实现农业环境多参数实时监测与可视化展示。

（2）实现设备远程智能控制，包括水阀、水泵、LED补光灯、通风风扇等执行器的开关控制与参数调节。

（3）实现基于阈值的预警规则配置与自动触发机制，保障农作物生长环境安全。

（4）实现传感器数据与设备动作的联动自动化控制，减少人工干预。

（5）实现智能灌溉功能，包含土壤湿度监测、灌溉量自动计算、灌溉操作控制及用水量统计。

（6）实现智能光照功能，包含光照强度监测、LED补光灯亮度精准调节及补光建议。

（7）扩展通风风扇设备接入，验证系统扩展性可行性。

### 2.2 工作内容

本项目的工作内容包括以下五个方面：

**（1）智能控制中心开发**：完成智能主机网关的功能适配，实现设备连接管理、指令转发、联动逻辑配置，确保APP与设备间通信稳定。

**（2）智能灌溉功能开发**：实现土壤温湿度数据采集与APP展示、灌溉水量自动计算的算法开发、灌溉操作控制及状态实时反馈功能。

**（3）智能光照功能开发**：完成光照强度数据采集与APP展示、LED补光设备亮度调节逻辑开发，确保光度调节精准可控。

**（4）智能环境监测功能开发**：实现多参数实时采集、APP可视化展示、阈值预警规则配置与预警触发功能。

**（5）扩展功能开发**：选定通风风扇作为扩展设备，完成设备接入接口开发、APP监测与控制功能开发，验证扩展性可行性。

## 4 数据库设计

### 4.1 数据库概述

系统采用MySQL 8.0关系型数据库，数据库名为`smart_agriculture`，字符集为`utf8mb4`。共设计12张数据表，涵盖传感器数据存储、设备管理、预警管理、联动控制、用户管理等功能。

### 4.2 数据表设计

#### 4.2.1 用户表（user）

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password_hash | VARCHAR(128) | NOT NULL | 密码哈希值 |
| role | VARCHAR(20) | DEFAULT 'user' | 角色（user/admin） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

#### 4.2.2 传感器数据表（sensor_data）

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 数据ID |
| temperature | FLOAT | NOT NULL | 环境温度（°C） |
| humidity | FLOAT | NOT NULL | 环境湿度（%） |
| light_intensity | FLOAT | NOT NULL | 光照强度（Lux） |
| co2_level | FLOAT | NOT NULL | CO₂浓度（ppm） |
| soil_moisture | FLOAT | NOT NULL | 土壤湿度（%） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 采集时间 |

#### 4.2.3 设备表（device）

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 设备ID |

#### 4.2.4 预警规则表（alert_rule）

| is_enabled | TINYINT | DEFAULT 1 | 是否启用 |

系统还包含以下数据表：

- **device_log**（设备操作记录表）：记录每次设备操作的动作、参数、来源（手动/自动）和时间。
- **scheduled_task**（定时任务表）：存储定时开关设备的计划任务，支持cron表达式和重复类型。
- **zone**（区域表）：管理温室大棚和田地区域信息。
- **zone_device**（区域设备关联表）：建立区域与设备的多对多关系。
- **crop**（作物表）：记录作物名称、品种、生长阶段、目标环境参数。
- **daily_report**（每日报告表）：存储每日环境数据均值、预警次数、灌溉次数、用水量等统计。
- **water_usage**（用水量记录表）：记录每次灌溉的用水量、持续时间。

## 5 非功能需求

### 5.1 可用性需求

（1）系统应7×24小时持续运行，传感器数据采集间隔不超过5秒。

（2）APP页面加载时间不超过2秒，设备控制指令响应时间不超过1秒。

（3）WebSocket连接断线后应在3秒内自动重连。

（4）网络异常时APP应降级为离线模式，展示缓存数据，不影响基本浏览。

### 5.2 安全性需求

（1）用户密码必须经过哈希加密后存储，禁止明文存储。

（2）API接口应支持基于Token的身份认证机制。

（3）后端应配置CORS跨域安全策略，限制非法来源请求。

（4）数据库连接信息应通过环境变量配置，不硬编码在源代码中。

### 5.3 兼容性需求

（1）前端APP应适配OpenHarmony/HarmonyOS NEXT系统，支持手机和平板两种设备形态。

（2）后端API遵循RESTful设计规范，支持跨平台客户端调用。

（3）数据库使用MySQL 8.0+，兼容主流Linux和Windows部署环境。

### 5.4 可扩展性需求

（1）设备类型应支持动态扩展，新增设备类型无需修改核心架构代码。

（2）预警规则和联动规则应支持用户动态配置，无需重启服务。

（3）系统采用模块化设计，各功能模块之间低耦合，便于独立开发和维护。

### 5.5 性能需求

（1）后端API平均响应时间不超过500ms。

（2）系统应支持至少10个WebSocket客户端同时连接。

（3）数据库应能存储至少100万条传感器历史数据，查询响应时间不超过1秒。
